# Цель работы

Получение практических навыков работы в консоли с атрибутами файлов, 
закрепление теоретических основ дискреционного разграничения доступа в современных системах с открытым кодом на базе ОС Linux.

# Выполнение

Создаем новую учетную запись guest

![Создаем учетную запись](Screens/Screenshot_1.PNG){ #fig:001 width=70% height=70% }

Задаем новый пароль для созданной учетной записи

![Задаем новый пароль](Screens/Screenshot_2.PNG){ #fig:002 width=70% height=70% }

Переходим в новую учетную запись.

![Переход в новую учетную запись](Screens/Screenshot_3.PNG){ #fig:003 width=70% height=70% }
 
Вводим команду pwd, смотрим находимся ли в домашней директории. Вводим команду whoami, чтобы узнать имя текущего пользователя. 
Вводим команду id и команду groups, сравниваем. На скриншоте видно, что команды показывют группу guest.

![Команды pwd,whoami,id,groups](Screens/Screenshot_4.PNG){ #fig:004 width=70% height=70% }

Просматриваем файл /etc/passwd.
 
![passwd](Screens/Screenshot_14.PNG){ #fig:005 width=70% height=70% }

Применяем команду с фильтром, чтобы получить информацию о пользователе.

![passwd с grep](Screens/Screenshot_5.PNG){ #fig:006 width=70% height=70% }
 
Определяем директории в home. Как мы видим установлены права для пользователей, на чтение запись и исполнение.

![ls -l /home/](Screens/Screenshot_6.PNG){ #fig:007 width=70% height=70% }

Смотрим атрибуты, просмотр атрибутов для основго пользователя не доступен, атрибуты же для директории guest не установлены.

![lsattr /home](Screens/Screenshot_7.PNG){ #fig:008 width=70% height=70% }
 
Создаем поддиректорию dir1 в домашней диреткории.

![mkdir dir1](Screens/Screenshot_8.PNG){ #fig:009 width=70% height=70% }

Смотрим права и атрибуты для dir1. Атрибуты не установлены, права доступа на чтение запись и выполнение.

![ls -l |grep dir1](Screens/Screenshot_9.PNG){ #fig:010 width=70% height=70% }

![lsattr |grep dir1](Screens/Screenshot_10.PNG){ #fig:011 width=70% height=70% }

Воспольземся командой chmode 000 dir1. Проверим правильность выполнения командой ls -l |grep dir1.

![chmod 000 dir1](Screens/Screenshot_11.PNG){ #fig:012 width=70% height=70% }
 
Попробуем создать файл в директории dir1. Проверяем наличие файла.

![echo "test" > /home/guest/dir1/file1](Screens/Screenshot_12.PNG){ #fig:013 width=70% height=70% }

![Проверяем создался ли файл](Screens/Screenshot_13.PNG){ #fig:013 width=70% height=70% }
